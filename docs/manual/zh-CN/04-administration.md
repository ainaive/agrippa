# 平台管理

管理分为两层：**组织管理**（侧边栏「管理」分组，仅组织管理员可见）负责共享注册表；**项目管理**（各项目的「设置」页，项目管理员可见）负责单个项目。

## 组织角色

组织角色有两种：**组织管理员**与**组织成员**。组织管理员管理下述注册表、邀请新成员、并可查询审计日志；但在项目内部*没有*隐式权限——项目成员身份永远是显式的。

## 账号与接入

自助注册已**关闭**——接入采用邀请制，实例不对任意注册开放。

- **首位管理员**——通过引导脚本在界面外创建：
  ```sh
  AGRIPPA_BOOTSTRAP_EMAIL=you@example.com \
  AGRIPPA_BOOTSTRAP_PASSWORD='设置一个强密码' \
  bun --env-file=../../.env.local apps/api/src/cli/bootstrap-admin.ts
  ```
  脚本对邮箱幂等，重复运行同一地址不会重复创建。密码用与登录流程一致的哈希算法存储，建好后即可在登录页登录。
- **邀请成员**——组织管理员打开 **管理 → 成员**，输入受邀人邮箱，生成一次性邀请链接（有效期 7 天）。由于 Agrippa 不内置邮件发送，请通过其他渠道把链接发给对方。受邀人打开链接、设置姓名与密码后即创建为**组织成员**——随后正常登录。待接受的邀请会列出并可在被使用前撤销。每个邀请令牌都只存哈希，因此仅数据库泄露无法兑现邀请。

## 注册表（管理分组）

每类注册表在侧边栏「管理」分组下各有独立页面，通过创建/编辑对话框维护——资源完全可以在界面中管理。

**模型** —— 模型提供商注册项：显示名、模型 ID、**档位**（`strong` / `balanced` / `fast`）、上下文窗口、每百万 Token 单价（用于成本核算与预算控制）。模板只请求档位、从不指名模型，所以升级模型阵容只需改注册表和项目授权——不动任何模板。模型可停用而无需删除。

**硅基人** —— 预置智能体（人设 + 系统提示词 + 头像），可就地编辑。可以在三位内置之外添加自定义角色。名称与人设需同时提供中英文。

**技能** —— 智能体可加载的打包指令集（Claude Code 技能格式）。平台内置 `git-workflow` 与 `test-runner`；每个技能有不可变的语义化版本（通过「新建版本」添加），模板以 `slug@范围` 引用（如 `builtin/git-workflow@^1`）。

**MCP 服务** —— Model Context Protocol 集成（如 GitHub）。注册传输方式（`stdio`/`http`/`sse`）与 JSON 配置；可选的**认证令牌只写不读**——录入后立即加密，界面此后只显示「已配置凭证」。编辑时留空表示保留现有令牌，用显式的移除开关才会清除。配置修改会递增修订号，供执行记录审计。

**模板** —— 所有编排模板及其版本历史。点击进入**编辑器**：编辑 YAML 源码、**校验**（一次性列出全部问题的试编译，并实时预览输入项将生成的提交表单）、**保存草稿**，再**发布**。已发布版本不可变——发布只是移动新提交所用的「最新」指针，进行中与历史执行仍使用各自固定的版本。平台自带模板在源码变更后会随部署自动重新发布。

## 项目设置（设置页，项目管理员）

- **常规** —— 重命名项目、编辑描述，并可在危险操作区（需确认）**归档**项目：项目从切换器中消失并停止接受新提交，历史执行与数据保留。
- **成员** —— 按邮箱添加（对方需已注册）、调整角色、移除（需确认）。项目必须至少保留一名管理员，平台会阻止降级或移除最后一名。
- **资源授权** —— 按资源类型的授权开关。这里是闸门：模板需要而这里未授权的资源，会让提交快速失败并给出具名错误。**可选**资源同样受此限制——可选的技能或 MCP 服务（例如「提交 PR」步骤所依赖的 GitHub 服务），未授权时不会提供给执行，也绝不会用共享凭证去解析。此时：明确要求该资源的步骤会被跳过，而仅仅「可以用到」该资源的步骤则会在没有它的情况下继续运行。
- **代码仓库** —— 项目执行可检出的 git 远端：地址、默认分支、可选访问令牌（只写、加密；仅在克隆时注入，智能体代码运行前即被清除）。
- **模型服务商** —— 项目级的模型服务商 API 密钥。选择服务商（例如运行通义千问的**阿里云百炼 / DashScope**），粘贴其 API 密钥（只写、加密——界面上永远只显示「已配置密钥」），并可为区域化部署覆盖接入地址（百炼国际站使用工作空间专属的 `*.maas.aliyuncs.com` 域名）。项目凭证会**覆盖**平台工作节点自身对应服务商的密钥；移除后回退到平台密钥。部分服务商（如百炼）没有平台级回退——若任务解析到其模型而项目未配置凭证，提交会以具名错误失败并指向此处。原地更换密钥即可轮换；进行中的执行会在下一个步骤使用新密钥。之后若要更改接入地址，必须在同一次操作中重新输入密钥——地址与密钥同行，已存储的密钥绝不会被重定向到新的主机。
- **配额** —— 每月费用（美元）和/或 Token 上限，附**强制停止**开关。强制配额耗尽后拒绝新提交、并在下一个步骤边界中止进行中的执行；非强制配额仅作提示。

## 用量与审计

各项目侧边栏的**用量**页展示当月花费与配额对比、Token 总量、每日花费图表，以及按模型和按任务类型的拆分；工作台保留核心数字。同样的数据可通过 `GET /api/v1/projects/:id/usage` 获取。

系统中的每一次变更——成员调整、资源授权、模板发布、任务提交、审批、取消——都会写入审计记录（操作者、动作、资源、内容、IP）。组织管理员可在管理分组的**审计日志**页浏览：按项目或动作筛选，展开任意记录查看内容详情。同样的数据也可通过 `GET /api/v1/audit-logs?projectId=&action=&limit=` 查询。
