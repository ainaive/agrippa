# 执行任务

## 提交

在侧边栏打开**任务目录**——任务类型按场景分组，且支持搜索。选择一个并填写表单。表单由模板的参数 Schema 自动生成——文本框、下拉、开关、仓库选择器都按模板作者的声明呈现；必填项以 `*` 标记。带代理位的模板（如**需求交付**）还会显示**代理**卡片：每个代理位（实现者、评审者……）一行，可覆盖的代理位允许更换人设与引擎（Claude Code / OpenAI Codex）——默认值是模板作者的选择。表单旁的**概要面板**显示执行者、将要执行的模板版本，以及本次执行的**预算**（费用与时长上限）——提交前请先确认。

提交可能被直接拒绝，且给出具体原因：所需技能或 MCP 服务未授权给该项目、没有已授权模型满足所需档位、参数校验失败、或强制配额已耗尽。到「设置」里解决后重新提交即可——被拒绝的提交不会留下任何数据。

## 观察执行

提交后会进入实时更新的**执行详情**页：

- **时间线**页签（默认）：以对话形式呈现整个执行——阶段标题（循环阶段标注「第 2/3 轮」）、每个代理带人设与引擎标识的流式输出、恰好出现在暂停之处的交互卡片、队友的评论，以及（交付类工作流的）分支、推送与最终的 **PR 卡片**（含打开按钮）。成员可在底部的评论框发言。
- **阶段**栏（左侧）：步骤按模板阶段分组（循环轮次带轮次角标），逐条显示状态、耗时、花费、尝试次数与所属代理位；检查点显示其决定状态。步骤可能显示*已跳过*（条件为假或可选集成不可用）——这是正常现象，不是错误。
- **预算**面板：花费相对费用上限、耗时相对时长上限的实时进度条，以及分阶段上限。
- **详情**面板：固定的模板版本、执行器，以及冻结的模型解析结果。页头显示各代理位的人设 + 引擎，以及平台创建的工作分支。
- **活动**页签：原始实时事件流——工具调用、子智能体启动、工作区检出、重试。
- **产出物**页签：交付物随产出即时出现并可内联预览——Markdown 直接渲染、补丁高亮着色、JSON 格式化、链接可点击；所有产出物均可下载。
- **参数**页签：本次执行使用的精确输入快照。
- 页头实时显示**花费**与总耗时。

关闭页面不会丢失任何内容——重新打开会完整回放历史事件并接续实时流。

## 响应检查点

执行暂停时，时间线（以及收件箱）中会出现琥珀色的**检查点卡片**。卡片内容取决于类型——任何项目*成员*或*管理员*都可以响应：

- **审批**呈现相关材料（如实现方案），提供**批准**/**驳回**与可选备注。循环内的方案确认还有**要求修改**：写下需要调整之处，代理会据此修订方案——最多到循环轮数上限，最后一轮只剩批准/驳回。
- **问题表单**渲染代理的澄清问题。每个问题都带代理的*建议答案*——一键采纳或自行作答；**全部采纳建议**可一次填完。若代理没有问题，执行会自动继续、不作停顿。
- **评审关卡**展示评审者的问题清单（含严重度与文件位置）。勾选需要实现者修复的项（**修复所选**）——未勾选的会被*明确接受*并作为豁免列入 PR 描述——或选择**全部接受并继续**跳过修复轮次。当评审结果干净时，关卡自动通过。

无人处理的检查点会在模板定义的超时后到期——通常是取消执行。

侧边栏的**审批**页是跨项目的**「等你处理」**收件箱：所有项目中待处理的检查点按类型标注（确认 / 回答问题 / 评审结果）并按项目分组，带实时数量角标。在任意条目上展开**审阅**即可当场响应——无需打开执行详情。

## 团队协作

项目内所有人看到同一条实时时间线；谁先响应检查点谁生效（后来者会收到友好的冲突提示）。已决定的检查点显示**由谁**在何时响应。成员可以在执行的任何时刻发表评论——提问、补充背景、「可以上了」——所有观看者都会即时看到。被接受的评审问题绝不会被悄悄略过：它们会连同接受者的名字出现在 PR 描述中。

对于交付类工作流，最终分支只包含一个由平台创建的快照提交，其文件树与已评审补丁完全一致。代理可以把本地提交当作工作检查点，但这些中间提交历史不会带入 PR。

## 取消、重试与失败

- **取消**（成员及以上）：排队/暂停中的执行立即取消；执行中的在下一个安全点停止。
- **重试**：出现在已结束的执行上——以相同参数创建新的执行（第 2、3 次……），固定在同一模板版本。
- **失败**总是带有原因：`budget_exceeded`（执行预算或项目配额）、`approval_rejected`、`contract_violation`（必需产出物未产出）、`timed_out`、或某步骤的错误。失败前已完成的步骤、输出与产出物仍然可见、可下载。
