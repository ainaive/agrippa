# 执行任务

## 提交

在侧边栏打开**任务目录**——任务类型按场景分组，且支持搜索。选择一个并填写表单。表单由模板的参数 Schema 自动生成——文本框、下拉、开关、仓库选择器都按模板作者的声明呈现；必填项以 `*` 标记。表单旁的**概要面板**显示执行者（Faber）、将要执行的模板版本，以及本次执行的**预算**（费用与时长上限）——提交前请先确认。

提交可能被直接拒绝，且给出具体原因：所需技能或 MCP 服务未授权给该项目、没有已授权模型满足所需档位、参数校验失败、或强制配额已耗尽。到「设置」里解决后重新提交即可——被拒绝的提交不会留下任何数据。

## 观察执行

提交后会进入实时更新的**执行详情**页：

- **时间线**（左侧）：步骤按模板的**阶段**分组，逐条显示状态、耗时、花费、尝试次数与执行它的模型角色；审批检查点内联出现在所属阶段中。步骤可能显示*已跳过*（条件为假或可选集成不可用）——这是正常现象，不是错误。
- **预算**面板：花费相对费用上限、耗时相对时长上限的实时进度条，以及分阶段上限。
- **详情**面板：固定的模板版本、执行器，以及冻结的模型解析结果（每个角色实际使用哪个模型）。
- **输出**页签：步骤执行期间的流式文本；完成后展示每步的最终输出。
- **活动**页签：本次执行的实时事件流——工具调用、子智能体启动、工作区检出、重试。
- **产出物**页签：交付物随产出即时出现并可内联预览——Markdown 直接渲染、补丁高亮着色、JSON 格式化、链接可点击；所有产出物均可下载。
- **参数**页签：本次执行使用的精确输入快照。
- 页头实时显示**花费**与总耗时。

关闭页面不会丢失任何内容——重新打开会完整回放历史事件并接续实时流。

## 审批

执行到达检查点时会暂停，出现琥珀色的**审批横幅**，标明检查点并呈现相关产出物（如拟定的修复方案）。任何项目*成员*或*管理员*都可以**批准**（执行从暂停处继续）或**驳回**（执行以 `approval_rejected` 失败），可附备注。无人处理的检查点会在模板定义的超时后到期——通常是取消执行。

侧边栏的**审批**页是跨项目收件箱：所有项目中待处理的检查点按项目分组列出，侧边栏入口带有实时数量角标。在任意条目上展开**审阅**即可查看呈现的产出物并当场批准或驳回——无需打开执行详情。

## 取消、重试与失败

- **取消**（成员及以上）：排队/暂停中的执行立即取消；执行中的在下一个安全点停止。
- **重试**：出现在已结束的执行上——以相同参数创建新的执行（第 2、3 次……），固定在同一模板版本。
- **失败**总是带有原因：`budget_exceeded`（执行预算或项目配额）、`approval_rejected`、`contract_violation`（必需产出物未产出）、`timed_out`、或某步骤的错误。失败前已完成的步骤、输出与产出物仍然可见、可下载。
