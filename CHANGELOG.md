## [Unreleased]

### Added

- **Per-project provider credentials (Aliyun Bailian/DashScope first)** ŌĆö projects configure model-provider API keys in Settings ŌåÆ Providers instead of relying on worker env ([ADR-0013](docs/adr/0013-per-project-provider-credentials.md)):
  - *Generic credential store.* One `provider_credentials` row per (project, provider) with an optional endpoint override; the key lives encrypted in `secrets` (new kind `provider_api_key`), write-only through the API (reads expose `hasCredential` only), rotated in place, deleted together with its secret, audited on every mutation.
  - *Provider catalog.* `PROVIDER_CATALOG` in `@agrippa/core` declares per-wire-protocol default endpoints and an auth policy per provider ŌĆö `dashscope` requires a project credential (no legitimate env fallback), `anthropic`/`openai` keep worker env as the deployment default. Qwen models (`qwen3.7-max/plus`, `qwen3.6-flash`) are seeded; claude-agent-sdk serves `dashscope` (claude-only for now ŌĆö Codex CLI Ōēź0.122 removed the chat wire API Bailian's compatible mode speaks; ADR-0013 amendment).
  - *Executor injection, project wins.* The engine materializes the project's key fresh per step (decrypted in the worker, redactor-registered before use, so rotation applies at the next step) and executors overlay it by replacing the wire protocol's entire auth-var family ŌĆö Bailian lands as bearer `ANTHROPIC_AUTH_TOKEN` + gateway URL on claude; explicit `openai` proxy overrides land as a synthesized `-c model_providers` entry with a per-run `CODEX_HOME` on codex (so ambient `auth.json` can't outrank the project key). Codex now registers on a CLI probe alone; env auth at boot is no longer required.
  - *Base-URL overrides are policy-checked, and endpoint + key travel together.* A credential's baseUrl is where the worker sends the decrypted key, so overrides must be https, carry no userinfo/query, point at a public DNS name (never an IP literal or localhost ŌĆö and the worker re-checks that the host doesn't *resolve* into private space at each step), and stay within the provider's pinned hosts (dashscope ŌåÆ the exact API hosts + `.maas.aliyuncs.com`, excluding customer-controlled OSS bucket domains) ŌĆö rejected as `base_url_invalid` otherwise. Changing the endpoint requires re-entering the key in the same request, so an existing write-only key can never be redirected; clearing back to the catalog default needs no key.
  - *Role-scoped, single-provider slot resolution.* Slots resolve only the model roles their steps (and subagents) reference, and provider-constrained slots resolve all of them from one provider (credentialed provider first, then cheapest, deterministic) ŌĆö a step's base URL is process-wide, so mixed-provider slots could never execute. A slot blocked only by a missing credential fails submission with the new `provider_credential_required` error ŌĆö retries re-assert the same gate against the frozen resolution, and a credential deleted mid-run fails the run with the same code before the executor is invoked; demo/`fake` resolution is unchanged.
  - *Keyless workers defer instead of failing.* Executors advertise which providers their worker env can authenticate (`envAuthProviders`); a run whose providers neither a project credential nor the claiming worker's env covers is declined before any status transition (the existing `run.deferred` heterogeneous-fleet path), so it waits for a capable worker rather than burning mid-run.

### Fixed

- **Fourth review round (PR #5)** ŌĆö the evidence boundary now excludes all agent-writable Git metadata:
  - *Platform Git owns a complete private gitdir and index.* Checkout gives the agent a copy of `.git` while the pristine original moves to a sibling sidecar. Evidence and publication rebuild the sidecar index from the trusted clone base and explicitly exclude `.claude`, `.mcp.json`, and `.agrippa`, so a config symlink/special file or agent edits to the index and `info/exclude` cannot alter the platform view. This supersedes the third round's config-restore design.
  - *Publication is one verified snapshot commit.* The platform compares stored and current patch evidence without truthiness shortcuts, treats Git failures as errors, refuses empty snapshots, stages once more inside the SCM operation, and publishes one idempotent Agrippa-authored commit whose tree matches the approved patch. Agent-local commits remain working checkpoints but are intentionally not preserved in the PR history (ADR-0012).
  - *Agent project configuration is invocation-scoped.* `.claude` and `.mcp.json` are removed without following symlinks before resource materialization and every agent attempt/resume, then only authorized skills are restored. Settings and hooks left by one agent cannot affect the next.
  - *Platform Git receives no model-provider credentials.* It now uses a system-only allow-list rather than the executor environment; hostile hooks/filters are still neutralized, and regression tests assert Anthropic/OpenAI/Codex keys never enter Git children.
- **CodeRabbit review pass (PR #5)** ŌĆö the publish snapshot commit is now fully deterministic (author/committer dates pinned to the clone base, so any retry or racer reproduces the identical SHA ŌĆö the `update-ref` CAS then chooses between equals); `request_changes` requires a non-empty comment at the schema level (the revision step interpolates it ŌĆö the UI already enforced this, the API now does too); checkpoint panels keep working when the evidence preview fails to load (a retry button appears, sending findings back for fixing stays possible, while accepting/approving blind ŌĆö including a partial fix's implicit waiver ŌĆö stays blocked); "Accept all recommendations" can no longer submit an answer set missing required answers; the run timeline follows the live stream instead of auto-scrolling only on mount; `run.deferred` timeline entries show the concrete reason; plus test-correctness fixes (awaited `rejects` assertions, temp-dir cleanup) and doc alignment (executor diagram, loop-only request-changes, slot-interchangeability limits ŌĆö both locales).
- **Third review round (PR #5)** ŌĆö five fixes; the platform now trusts nothing an agent can write:
  - *Platform git no longer executes agent-controlled hooks with worker secrets.* Every platform git call (evidence diffs, the finalizing commit, the credentialed push) runs in a directory the agent could write ŌĆö including `.git/**` ŌĆö and used to inherit the worker's full environment. The `git()` wrapper now uses the executor allow-list env scrub (no `DATABASE_URL`/`AGRIPPA_SECRET_KEY`/provider keys reach any git child), disables global/system gitconfig and neutralizes `core.hooksPath`/`core.fsmonitor` on every call, and restores repo-local `.git/config` from a provision-time snapshot before diff/push ŌĆö erasing agent-added filters, textconv drivers, `url.insteadOf` push redirection, and credential helpers. The snapshot and the clone-base SHA live in a per-run platform sidecar outside the agent-writable tree, which also stops an agent from moving `refs/agrippa/base` to blank the patch evidence. A hostile-workspace test drives the publish path with weaponized hooks, filters, and an `insteadOf` redirect and asserts none of it executes.
  - *Reviewers are read-only and drift refuses to publish.* v2 agent steps take a per-step `access:` override; the delivery template's review step declares `readOnly` (claude enforces it via the isolation seam, codex via `--sandbox read-only` with fenced-json artifact synthesis). The round-2 push-time evidence *refresh* is replaced by a hard failure: `git.push` fails the run (`contract_violation`) when the workspace no longer matches the stored patch ŌĆö refreshed evidence is not approved evidence.
  - *Lost workspaces fail fast.* Workspaces are host-local; a resume landing on a worker host without the checkout now fails with `workspace_lost` instead of silently running every remaining step against an empty directory (re-provisioning would fabricate a workspace missing the work branch and all agent commits ŌĆö deliberately not attempted).
  - *Duplicate interaction ids are rejected.* Questions and review findings are keyed by id everywhere downstream (answers, selections, waivers, React keys); the artifact schemas now reject duplicates.
  - *ADR hygiene.* ADR-0010's semantics correction is recorded as an appended amendment (the append-only rule applies even pre-merge), and ADR-0011 gains an addendum for the platform-git-distrust and read-only-reviewer decisions.
- **Second review round (PR #5)** ŌĆö six fixes, four of them establishing one evidence contract: *the patch artifact a user approves and the pushed PR contain exactly the same changes, and the sanitized paths (`.claude`, `.mcp.json`, `.agrippa`) appear in neither, in either direction*:
  - *PRs can no longer silently delete sanitized repository files.* The round-1 sanitization `rm`'d repo-supplied `.claude`/`.mcp.json` and hid the deletions only from the diff ŌĆö an agent's `git add -A && commit` still shipped them. Sanitized paths are now invisible to git itself: tracked entries get `--skip-worktree` before removal and the paths join `.git/info/exclude` (covering the *untracked* side, which the platform itself populates ŌĆö materialized skills under `.claude/skills/`, artifact scratch under `.agrippa/`), so the diff's pathspec excludes are gone entirely and evidence and commits agree by construction.
  - *Patch evidence now equals the push.* `git.push` gains a platform finalizing commit for anything the agent left uncommitted (Agrippa identity, host gitconfig isolated), refuses to publish a branch with no commits and no changes ("nothing to publish" instead of a provider 422 later), and re-diffs at push time to detect post-review drift. (The refresh-and-publish behavior this round introduced was replaced in the third round: drift now fails the run.)
  - *`{}` no longer reads as a clean review.* The interaction artifact schemas are strict with required lists: a missing or typo'd `findings`/`questions` key previously parsed as an empty artifact (zod default + unknown-key stripping) and auto-passed the gate; it now fails the producing step. Auto-pass stays asymmetric by design: absent questions pass (nothing to ask), absent review reports fail (a gate without evidence never passes).
  - *Heterogeneous fleets defer instead of burning retries.* A worker picking up a run bound to an executor it didn't register now declines the job before any status transition (typed `ExecutorUnavailableError`), appends a `run.deferred` timeline event, and lets the reconciliation sweepers re-enqueue until a capable worker claims it; `running` runs still rethrow (execution lease remains future work).
  - *Work-branch suffixes widen from 32 to 48 random bits* (12 hex chars of the UUIDv7 tail) ŌĆö ~1.2% birthday-collision odds at 10k runs was too high for a scheme `pr.open`'s duplicate-recovery depends on.
  - *Docs match the runtime again* ŌĆö ADR-0010 and the design docs now state the asymmetric auto-pass semantics and the heterogeneous-worker recovery path.
- **Review-round hardening (PR #5 code-review findings)** ŌĆö eight fixes, four of them blockers for the requirement-delivery workflow:
  - *Committed work no longer vanishes from patch artifacts.* Checkout pins the clone-time HEAD as `refs/agrippa/base` and `diff()` reports against it (committed + staged + worktree, with the sanitization deletions excluded); a required patch whose auto-diff comes back empty now fails the producing step retryably ŌĆö before push/PR, not at end-of-flow. Also repairs the latent empty-patch bug in v1 `bug-localize-fix`. New real-git integration tests cover exactly the path the fake-workspace compliance fixtures could not.
  - *Malformed interaction artifacts no longer auto-pass their checkpoints.* Questions/review-report artifacts are validated against the shared schemas at store time (the producing step fails with `contract_violation`; template retry applies ŌĆö the review step gains one), with a strict checkpoint-time backstop for resumed runs: an absent review report fails the run (a gate without evidence never passes), and over-inline-limit sources get a distinct error instead of reading as "no findings".
  - *Work branches are unique across tasks.* Run numbers restart per task, so the default branch name appends the run id's random tail (`agrippa/run-<n>-<shortId>`, exposed as `run.shortId` ŌĆö the UUIDv7 *tail*; its head is timestamp bits).
  - *Submission rejects executors the deployment cannot run.* Workers advertise their registered executors in a heartbeated `executor_registrations` table (migration 0006); `resolveAgentBindings` rejects any resolved binding without a recent registration (`executor_unavailable`, checked before model resolution), and the submit page's picker disables unavailable engines. Heterogeneous workers can still race at pickup; the second review round above turns that into a deferral instead of burned retries.
  - *Checkpoint panels present the latest evidence.* The artifacts endpoint now orders rows and exposes `iteration`, and panels take the last row per key ŌĆö after "request changes", responders can no longer approve the previous round's plan or patch.
  - *`pr.open` is idempotent across retries.* A GitHub 422 / GitLab 409 on the retry recovers the existing open PR by head/base (safe now that branches are unique) instead of failing a run whose PR exists.
  - *Codex cannot load undeclared capabilities.* `codex exec` runs with `--ignore-user-config` and `--ignore-rules` (auth still resolves from `CODEX_HOME`), and the boot probe refuses to register a CLI that lacks those flags ŌĆö silently running without config isolation is the vulnerability.
  - *Question contracts are enforced end to end.* Select questions must carry options (a required optionless select was an unanswerable form ŌĆö input checkpoints have no reject, so runs deadlocked until timeout), recommendations must match the question kind (booleans allowed), and the respond endpoint validates every answer against its snapshotted kind and options.

### Added

- **Requirement-delivery workflow** ŌĆö the new `swdev.requirement-delivery` task type takes a natural-language requirement all the way to a reviewed pull request: a clarify loop where the implementer asks structured questions (with recommended answers; auto-continues when it has none), a plan loop whose approval supports *request changes* with the comment fed back into the revision, implementation on a platform-created branch, a cross-agent **review-fix loop** (implementer = Claude Code / Forge, reviewer = OpenAI Codex / the new **Arbiter** faber) where each round the user picks findings to fix or accepts the rest (zero findings auto-pass), and a publish phase that pushes and opens the PR platform-side ŌĆö with any accepted findings listed in the PR body as explicit waivers, including who accepted them. An extra sign-off appears only when the loop exhausts right after a fix that was never re-reviewed.
- **`agrippa/v2` template format** (ADR-0010) ŌĆö agent slots (named implementer/reviewer roles binding a faber + executor, user-overridable at submit with capability and provider checks), checkpoint *steps* in three kinds (`approval` incl. request-changes, `input` question forms ŌĆö auto-passing when the questions artifact is absent or validly empty ŌĆö and `review-gate` findings decisions, which auto-pass **only** on a present, schema-valid report with zero findings; absent review evidence fails the gate), bounded `loop` nodes (static `maxIterations`, `until` conditions), platform SCM system actions (`git.branch`/`git.push`/`pr.open`), and new expression roots (`checkpoints.<id>`, `artifacts.<key>`). v1 templates keep working unchanged through a pure compile-time upgrade; the whole engine compliance suite passes through it, plus new suites for slots, checkpoints, loops, and SCM.
- **OpenAI Codex executor** (`packages/executor-codex`, ADR-0011) ŌĆö wraps `codex exec --json` (event shapes pinned against codex-cli 0.145.0): normalized event mapping with cache-aware usage splitting, containment via Codex's native OS sandbox (read-only / workspace-write, network off, no interactive escalation), env through the shared allow-list scrubber, and fenced-json artifact synthesis for read-only reviewer steps. Registered only when the CLI probe and auth succeed; every registered executor is asserted against the new static `EXECUTOR_CATALOG` in core at worker boot.
- **Checkpoint interaction API + team collaboration** ŌĆö `POST /runs/:id/checkpoints/:checkpointId/respond` (kind-discriminated: decision/answers/finding selections, validated against the pending row's snapshot; CAS + event + audit in one transaction), a generalized cross-project `GET /checkpoints/pending` "waiting on you" inbox, and **run comments** (`GET`/`POST /runs/:id/comments`) that commit together with their `comment.added` run event so the live timeline and the thread can never disagree. Task submission accepts per-slot `agents` overrides; model resolution is per-slot and provider-filtered (a Codex slot without a granted OpenAI model fails at submit, not mid-run). The legacy `/approvals` decide routes are removed.
- **Run timeline UX** ŌĆö the run detail page's default tab is now a conversational timeline built from the replayed SSE stream: phase headers with loop-round chips, streaming agent turns tagged with each slot's persona and engine, inline checkpoint cards (question forms with one-click recommendations; findings tables with severity badges and explicit accept-remaining confirmation; request-changes on loop approvals), teammate comments with a pinned composer, and a PR card at the end. The submit page gains agent-slot pickers; the phases rail, budget/meta cards, and inbox are round- and kind-aware. OpenAI codex model rows join the seed (verify ids/pricing per deployment).

### Changed

- **`approvals` table ŌåÆ `checkpoints`** (data-preserving rename) with `kind`, `iteration`, and a structured `response`; `run_steps` and `artifacts` gain `iteration`; `runs` gain `agent_bindings` and `work_branch`; new `run_comments` table (migration `0005_requirement-delivery`). Engine events rename `approval.required/decided` ŌåÆ `checkpoint.required/decided` and add `loop.*`, `branch.*`, `pr.opened`, `comment.added`.
- **Demo mode hardening** ŌĆö when `AGRIPPA_EXECUTOR=fake`, every agent slot binds to the demo executor (a demo install must never silently route a slot to a real, key-consuming engine), `AGRIPPA_SCM=fake` fabricates branch/push/PR, and the demo executor is loop-round-aware (questions/findings in round 1, clean in round 2) so the delivery workflow demos end-to-end without tokens.

- **Invite-only onboarding; self-registration closed** ŌĆö `POST /api/auth/sign-up/*` is now guarded and returns `403 registration_closed` (localized). The only ways to join the org are the new bootstrap-admin script (`bun --env-file=../../.env.local apps/api/src/cli/bootstrap-admin.ts`, reads `AGRIPPA_BOOTSTRAP_EMAIL`/`PASSWORD`, idempotent, creates the first `org_admin`) and the invitation flow: `org_admin` calls `POST /api/v1/invitations` to get a one-time `?token=ŌĆ”` link (token stored hashed ŌĆö sha256 ŌĆö so a DB leak can't be replayed); the invitee opens `/accept-invite?token=ŌĆ”` (public, no session), sets name + password, and is created as `org_member` via direct `users`+`accounts` insert (`hashPassword` from `better-auth/crypto`), then signs in through the normal endpoint. `GET /api/v1/invitations` lists, `DELETE /api/v1/invitations/:id` revokes. New `invitations` table + migration `0004_invitations`. The SPA login page drops the sign-up mode; a new `/accept-invite` page and an Admin ŌåÆ Members invite-management page are added. No email infrastructure exists ŌĆö invite links are returned to the admin to share out-of-band. The `user.create` first-userŌåÆadmin hook is retained as a safety net but superseded by `bootstrap-admin`.
- **`APT_MIRROR` build arg for the worker image** ŌĆö `infra/Dockerfile.worker` now accepts an optional `APT_MIRROR` build arg that swaps Debian apt to a closer mirror before `apt-get install` (the worker image needs git/ripgrep/bubblewrap). `docker-compose.yml` threads it through as `build.args.APT_MIRROR: ${APT_MIRROR:-}`, documented in `infra/env/.env.example`. No-op when unset; set `APT_MIRROR=https://mirrors.aliyun.com` (or similar) for hosts where `deb.debian.org` is slow or blocked.

### Security

- **Executor isolation seam** ŌĆö one enforceable place (`packages/executor-core/isolation.ts`) decides every tool call and scrubs the subprocess environment. Read-only workspaces now actually deny shell and confine writes to the artifact directory; read-write workspaces confine writes to the workspace with a boundary-safe check (the previous `startsWith` let a sibling `<workspace>-evil` path through, and `Bash` bypassed the check entirely). **Reads (Read/Grep/Glob) are confined to the workspace too**, so the agent can't read `/proc/self/environ`, another run's directory, or the shared artifact store. The SDK subprocess environment is **allow-listed** ŌĆö only the SDK auth variables and a fixed set of system essentials pass through, so platform secrets, DSNs, and injection vectors like `NODE_OPTIONS` are all dropped ŌĆö with the OS `sandbox` enabled where available (bubblewrap installed in the worker image), `strictMcpConfig`, and repo-supplied `.claude`/`.mcp.json` removed after checkout so a checked-out repository can't inject hooks or permission overrides. The worker image runs as a non-root user with `/app` kept root-owned.
- **Event-payload secret redaction** ŌĆö known secret values (the provider key, resolved MCP tokens) are redacted from every event before it is persisted or streamed over SSE, so a secret the agent echoes into output can't leak through the timeline.
- **Artifact path containment** ŌĆö artifact ingestion resolves sources through `realpath` and rejects any that escape the workspace, closing a symlink disclosure (e.g. `ln -s /proc/self/environ`) that could exfiltrate secrets or other runs' files through the download endpoint.
- **Known dev-only advisory, accepted:** `bun audit` reports GHSA-67mh-4wv8-2f99 (esbuild dev-server request forgery) against esbuild 0.18.20, pulled in transitively by `drizzle-kit` ŌåÆ the deprecated `@esbuild-kit/esm-loader` ŌåÆ `@esbuild-kit/core-utils`. It does **not** affect shipped code: `drizzle-kit` is a devDependency, and the advisory concerns an esbuild dev server that `drizzle-kit` never starts (it already ships `tsx`, leaving the `@esbuild-kit` path vestigial). `drizzle-kit` 0.31.10 is the latest release, so no upgrade resolves it, and Bun supports neither nested `overrides` nor nested `resolutions` ŌĆö a *global* esbuild override would force Vite's and tsx's esbuild (0.28.1) down to 0.25.x and break both. Revisit when `drizzle-kit` drops `@esbuild-kit` or Bun gains scoped overrides.
- **Cross-tenant resource authorization** ŌĆö submission rejects a `repoConnectionId` that isn't owned by the project, and the worker loads repo connections scoped to the run's project. Optional skills/MCP servers are now grant-checked: an authorized-resource manifest is pinned onto the run at submit (required grants enforced, optional resources included only when granted) and the worker resolves resources only from it, so a project without a grant can no longer receive the platform's global credential (e.g. the shared GitHub token).

### Fixed

- **Main-branch CI red on PR merges** ŌĆö commitlint parsed the `Merge: ŌĆ” (#N)` merge-commit title as an invalid commit type, failing every post-merge push run (and skipping Test/Build with it). Merge subjects are now exempt from commitlint; the PR's own commits are still linted by the PR-range step.
- **Codex review findings** ŌĆö approvals can no longer be decided before their presented artifacts load (Approve stays disabled on load failure while Reject remains available, and unproduced `present[]` keys render explicitly); archiving a project no longer redirects straight back into it (active-membership filtering + clearing the stored last-project id); the approvals inbox shows a retryable error instead of "nothing pending" during an outage; sidebar links close the mobile drawer; and the usage chart derives its day domain and month label from the database clock that groups the data, not the browser's.
- **PR review findings** ŌĆö the run timeline shows the pinned phase plan on queued runs instead of "no steps"; usage's by-task-type grouping keys on task-type id (names can collide) and its empty check considers cost; Usage/RunDetail/TemplateEditor show a retryable error state instead of a blank page when their query terminally fails; icon-only controls gained aria-labels; template deprecation is conditional on the published status (race guard); and the screenshot harness no longer skips cleanup on failure (`process.exit` inside `try`), uses a per-run scratch database, and watches the login page for errors too.
- **Root `bun run dev`** ŌĆö the one-command dev boot crashed api/worker with "DATABASE_URL is not set": `bun --filter` runs each workspace's dev script with the workspace as cwd, outside the repo-root `.env.local`'s automatic loading. The api/worker dev scripts now pass `--env-file=../../.env.local` (a silent no-op when the file doesn't exist, so shell-export workflows are unaffected; exports still take precedence).

- **Dev-mode sign-in through the Vite proxy** ŌĆö better-auth trusts only its `baseURL` origin (`http://localhost:3000`), so auth POSTs from the dev SPA on `:5173` were rejected with 403 `INVALID_ORIGIN`. The dev proxy now rewrites the `Origin` header to the API's own, completing the same-origin simulation of the production deployment (ADR-0001) instead of loosening the API's origin check.
- **Crash recovery for no-retry steps** ŌĆö a worker that died mid-step no longer silently skips (or spuriously fails) a step without template retries; the crashed attempt no longer consumes the retry budget, and the executor session is carried onto the recovery attempt so resume works.
- **Atomic run lifecycle** ŌĆö one `finalizeRun` owns every terminal transition: a single transaction does the status CAS (requiring `cancel_requested = false` for a success, so a late cancel wins atomically), finishedAt/totals, and the terminal event. Both the engine and the worker's retry-exhaustion path use it, so a run is never half-finalized and a queued run whose setup threw transitions `queued ŌåÆ failed` (now legal) with a terminal event instead of stranding. Event sequence numbers come from an atomic per-run counter (`runs.next_event_seq`), and approval decisions are CAS on `pending` with the sweeper re-enqueuing any run left paused by a lost resume enqueue.
- **Quota accounting** ŌĆö the engine now counts the same monthly window as the submit gate, excludes the run's own spend from the headroom it checks (no double-count on resume), re-reads project usage at each step boundary so concurrent runs can't jointly overspend, and restores per-phase spend on resume so per-phase budgets aren't reset by a crash.
- **Artifact output contract** ŌĆö patch steps no longer hand-write the diff (the engine generates it from `git diff`); collected artifacts are validated against the step's declared keys/kinds and matched by exact filename; only the current step's own expected files are cleared before an attempt (never a recursive `rm` a `.agrippa -> /work` symlink could redirect at the shared store, and `.agrippa` is stripped at checkout); missing/empty sources don't create zero-byte rows; binary (`file`-kind) artifacts are stored byte-exact; and ingestion size-caps and streams files instead of buffering them whole.
- **Authorized resources** ŌĆö `requires.skills` is validated by the compiler and enforced by the engine; `skills` resolution returns `{ resolved, missing }` like `mcpServers`, so an unavailable required skill fails the step and an unavailable optional one is skipped (rather than throwing). `scripts/backfill-manifest.ts` reconstructs the manifest for pre-migration runs from **project grants** (not the full template).
- **SSE live gap** ŌĆö the events stream subscribes and *awaits* the subscription being live before replaying, and treats the bus purely as a **wake-up** that triggers an ordered Postgres replay ŌĆö so the cursor advances contiguously and a dropped event can't be skipped past (even on `Last-Event-ID` reconnect) (ADR-0007).
- **Production Compose** ŌĆö split into a worker-only `workspaces` volume and a shared `artifacts` volume; the `api` runs as the non-root `bun` user and mounts only `artifacts` (consistent ownership whichever service initializes it) and receives `AGRIPPA_EXECUTOR` (the API chooses the executor at submit).
